    2. E.g.: # This is my First Comment in Python
  2. Multi line Comments — E.g.: # I can write a multi-line comment
    # by using '#' in the starting of every line.
    # I’m Learning Python after learning java.
    3. OR
    E.g.: """
    I can write the above multi-line comment with the use of """ (Triple double quotes) or ''' (Triple single quotes) in the beginning and ending of the block
    """
    OR
    E.g.: '''
    I can write the above multi-line comment with the use of """ (Triple double quotes) or ''' (Triple single quotes) in the beginning and ending of the block
    '''

    * Multi-line comments are also known as block comments.

Docstring

Python supports only single line comments. Multi line comments are not available in Python. It means """ and ''' are not multi-line comments.

So, using """ and ''' are not recommended for comments since they internally occupy memory and would waste time of the interpreter since the interpreter has to check them.

The use of """ and ''' is to create an API (Application Programming Interface) document file. An API file is a text or HTML file that contains description of all the features of software, language or a product. When a new software is created it is the duty of the programmer to provide an API for that software. It is like a user manual or Help file.

If we write """ or ''' as first statements in a module, then these strings are called documentation strings or docstrings.
